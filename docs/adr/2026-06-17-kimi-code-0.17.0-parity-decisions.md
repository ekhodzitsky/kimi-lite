# ADR: Kimi Code 0.17.0 parity decisions

## Status

Accepted

## Context

Kimi Code CLI released `0.17.0` on 2026-06-17. This ADR records which
changes from that release are ported to `kimi-lite` and which are
explicitly out of scope for the lightweight Go CLI.

## 0.17.0 changes reviewed

Source: `@moonshot-ai/kimi-code@0.17.0` release notes on GitHub.

### 1. Server-hosted web UI (`kimi server`, `kimi web`)

**Decision:** Not ported.

`kimi-lite` is positioned as a fast, terminal-first native CLI. Adding a
local HTTP/WebSocket server and a browser-based chat client would
fundamentally change the product surface and require a large ongoing
maintenance burden (frontend build, session synchronization between TUI
and web, authentication for the local server, etc.). This conflicts with
the "lite" goal.

### 2. Show underlying OAuth token refresh connection errors

**Decision:** Not ported.

`kimi-lite` does not implement OAuth login or token refresh. Provider
configuration uses API keys via `config.toml`/environment variables, so
this patch has no equivalent code path.

### 3. Skip debug TPS when output stream is too short

**Decision:** Not ported.

This is an internal telemetry/debugging metric in Kimi Code. `kimi-lite`
does not expose tokens-per-second diagnostics, so the change is not
applicable.

### 4. Restore turn counter from persisted loop events on resume

**Decision:** Ported.

Although `kimi-lite` uses random turn IDs, turns are ordered by
`started_at`. To guarantee stable, monotonic turn numbering across
session resume and to avoid any possibility of reusing turn identifiers,
we added a persisted `seq` column to the `turns` table. `TurnManager`
queries `NextTurnSeq` from the store when starting a turn and assigns the
next value, continuing the counter from the highest persisted sequence.

See implementation in:

- `pkg/api/types.go` — `Turn.Seq` field and `TurnStore.NextTurnSeq`.
- `internal/store/migrations/003_turn_seq.sql` — schema migration.
- `internal/store/sqlite.go` — `SaveTurn`, `GetTurns`, `NextTurnSeq`.
- `internal/core/turn.go` — `startTurn` assigns `Seq` before persisting.

### 5. Fix web login page and no-workspace conversation startup flow

**Decision:** Not ported.

This fix is specific to the web UI and OAuth onboarding flows added in
0.17.0, neither of which exist in `kimi-lite`.

## Consequences

- `kimi-lite` remains a terminal-only tool.
- Resume correctness is improved by monotonic turn sequence numbers.
- The `api.Store` interface gained `NextTurnSeq`, which is a breaking
  change for third-party store implementations (mirrors the previous
  `TurnManager` interface expansion and is acceptable for a pre-1.0
  project).
