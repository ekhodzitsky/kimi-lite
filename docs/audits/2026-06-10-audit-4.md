## 🔴 P0 — Критические (8 штук)

### 1. Type assertions без `ok` check — panic на каждый keystroke
- **File:Line:** `internal/tui/model.go:294, 298, 303, 311, 316, 321, 329`
- **Description:** `m.input = inpModel.(*input.Model)`, `m.vp = vpModel.(*viewport.Model)`, `m.sidebar = sbModel.(*sidebar.Model)`, `m.messages[i] = updated.(*msgcomp.Message)`. Все type assertions — hard cast без comma-ok. Если child component вернёт другой тип (library update, test double, corrupted state) — instant panic.
- **Impact:** Каждый keystroke проходит через эти assertions. Любая несовместимость = crash.
- **Recommendation:** Заменить на safe type switch: `switch m := inpModel.(type) { case *input.Model: ... }`.

### 2. `glamour.Render` без `recover` — panic на malformed markdown
- **File:Line:** `internal/tui/messages/messages.go:214`
- **Description:** `glamour.Render` вызывается напрямую в `View()` path. Glamour использует Goldmark → Chroma → Bluemonday, известные panic paths на edge-case input (malformed tables, specific emoji sequences). Нет `defer recover()` в TUI.
- **Impact:** LLM возвращает malformed markdown → entire TUI process dies.
- **Recommendation:** Оборачивать `glamour.Render` в `recover()`, или выполнять в background goroutine.

### 3. `--config` CLI flag полностью сломан
- **File:Line:** `cmd/kimi-lite/main.go:83-84`, `internal/config/loader.go:92-95`
- **Description:** `loader.SetFlag("config", f.configPath)` делает `l.v.Set("config", f.configPath)` — устанавливает viper key "config", а НЕ путь к config file. Viper требует `SetConfigFile()` для указания пути к файлу.
- **Impact:** `--config /path/to/config.toml` полностью игнорируется. Viper ищет файл в стандартных путях.
- **Recommendation:** Добавить `SetConfigFile` в `Loader` и вызывать его для `--config`.

### 4. ContextCompressor уничтожает данные при partial failure
- **File:Line:** `internal/core/compressor.go:64-82`
- **Description:** `Compact` вызывает `store.ClearMessages(ctx, sessionID)` первым, затем по одному `AppendMessage` для summary и recent messages. Если `ClearMessages` прошёл, но любой `AppendMessage` падает (disk full, SQLite BUSY, process killed) — сессия остаётся с **нулём сообщений**.
- **Impact:** Перманентная необратимая потеря истории разговора.
- **Recommendation:** Обернуть в SQLite транзакцию: `BEGIN; DELETE ...; INSERT ...; COMMIT;`.

### 5. `resolveEnvVar` передаёт literal `$VAR` при пустой env var
- **File:Line:** `internal/config/loader.go:97-107`
- **Description:** `if val := os.Getenv(name); val != ""` — если переменная существует, но пустая, возвращается исходная строка `$MOONSHOT_API_KEY`.
- **Impact:** Пользователь получает cryptic auth failure, потому что literal `$MOONSHOT_API_KEY` отправляется в API.
- **Recommendation:** `if val, ok := os.LookupEnv(name); ok { return val }`.

### 6. SSRF redirect bypass — security theater (повтор с аудита 3)
- **File:Line:** `internal/app/app.go:62-70`, `internal/core/tools.go:55-68`
- **Description:** `newSecureHTTPClient()` с `isBlockedHost` существует, но `app.go` создаёт свой insecure client и передаёт его в `NewBuiltInToolExecutor`. Production path НИКОГДА не использует secure client.
- **Impact:** Полный SSRF через 302 redirects.
- **Recommendation:** Удалить custom client из `app.go`, передавать `nil`.

### 7. `go.mod` 1.25.0 vs CI 1.23 — fake fix (повтор с аудита 3)
- **File:Line:** `go.mod:3`, `.github/workflows/ci.yml:21`
- **Description:** Агент Wave 2 заявил исправление, но файл на диске всё ещё 1.25.0.
- **Impact:** CI полностью сломан.
- **Recommendation:** Понизить go.mod до 1.23 или поднять CI до 1.25.

### 8. `mustMarshalParams` вызывает `panic()`
- **File:Line:** `internal/core/tools.go:241-244`
- **Description:** `panic()` в production path. Хотя текущие static maps безопасны, любое динамическое значение (NaN, func, chan) вызовет panic.
- **Impact:** Latent crash point.
- **Recommendation:** Вернуть ошибку вместо panic.

## 🟠 P1 — Высокий приоритет (14 штук)

### 9. 429 Too Many Requests не retry-ится
- **File:Line:** `internal/llm/client.go:242-249`
- **Description:** `doRequest` retry-ит только 5xx и network errors. Все 4xx считаются fatal. HTTP 429 (rate limit) — transient error, должен retry-иться.
- **Impact:** Ненужные падения при rate limiting.
- **Recommendation:** Добавить 429 в retryable errors с backoff.

### 10. Store methods загружают unbounded данные без pagination
- **File:Line:** `internal/store/sqlite.go:115-138, 176-206, 253-294`
- **Description:** `ListSessions`, `GetMessages`, `GetTurns` — `SELECT ... ORDER BY ...` без `LIMIT`. Сессия с 10k сообщений загрузит всё в память.
- **Impact:** OOM на больших сессиях.
- **Recommendation:** Добавить `LIMIT` + `OFFSET`, или курсор. Enforce `MaxHistory`.

### 11. `write_file` / `str_replace_file` без size limits
- **File:Line:** `internal/core/tools.go:305-319, 321-345`
- **Description:** Нет проверки размера content. LLM может записать multi-GB файл.
- **Impact:** Disk exhaustion.
- **Recommendation:** Max file size limit (например, 10MB).

### 12. `shell` tool без limit на output size
- **File:Line:** `internal/core/tools.go:386-404`
- **Description:** `cmd.CombinedOutput()` буферизует весь stdout+stderr. `yes` или `find /` → OOM.
- **Impact:** Memory exhaustion.
- **Recommendation:** Использовать `io.LimitReader` на stdout/stderr.

### 13. Composite tool executor silently shadows tools
- **File:Line:** `internal/core/composite_tools.go:18-28`
- **Description:** При merge tool definitions duplicate names перезаписывают предыдущие без warning.
- **Impact:** MCP server может shadow built-in tools.
- **Recommendation:** Log warning или error на collision.

### 14. `/clear` во время streaming вызывает race
- **File:Line:** `internal/tui/model.go:523-566, 583-626`
- **Description:** Slash commands разрешены в любом state. `/clear` во время `TurnStreaming` стирает TUI, но `run()` goroutine продолжает append messages.
- **Impact:** Сообщения reappear после clear.
- **Recommendation:** Блокировать `/clear` при `state != TurnIdle`.

### 15. Tool result message IDs могут collide
- **File:Line:** `internal/core/turn.go:226`
- **Description:** `ID: result.CallID`. Если LLM reuse CallID — PRIMARY KEY conflict в SQLite.
- **Impact:** Turn aborts с constraint error.
- **Recommendation:** `ID: idgen.GenerateID()`, сохранять `CallID` в отдельном поле.

### 16. Нет валидации config после loading
- **File:Line:** `internal/config/loader.go:60-90`, `internal/config/raw_config.go:66-132`
- **Description:** Ни одно поле не валидируется: negative timeouts, empty model, malformed URL, negative MaxTurns.
- **Impact:** Silent misbehavior.
- **Recommendation:** `Validate()` метод с проверками.

### 17. `saveTurn` errors игнорируются в 5 из 6 call sites
- **File:Line:** `internal/core/turn.go:134, 164, 273, 356, 372`
- **Description:** Только `RunTurn:110` обрабатывает ошибку. Остальные — `_ = tm.saveTurn(...)`.
- **Impact:** Database corruption или disk full → silent data loss.
- **Recommendation:** Log errors или propagate.

### 18. `doRequest` и `doRequestStream` — 30 строк дублирования
- **File:Line:** `internal/llm/client.go:203-255, 259-309`
- **Description:** Идентичный retry loop, headers, status checking. Любое изменение — в двух местах.
- **Impact:** Maintenance nightmare, divergence risk.
- **Recommendation:** Extract shared helper.

### 19. `renderedContent` rebuild loop продублирован 3 раза
- **File:Line:** `internal/tui/model.go:645-650, 675-680, 927-935`
- **Description:** Один и тот же O(n) rebuild loop в трёх местах. Комментарий на строке 153 лжёт: "O(1) viewport updates".
- **Impact:** O(n) на каждый chunk, resize, message addition.
- **Recommendation:** Extract helper + incremental updates.

### 20. `context.Background()` в production hot paths
- **File:Line:** `internal/app/app.go:87`, `internal/mcp/tool_executor.go:24`, `internal/tui/model.go:607`
- **Description:** `context.Background()` вместо propagated context. MCP connect blocks startup. Tool enumeration не cancellable.
- **Impact:** Startup hangs, SIGINT не прерывает.
- **Recommendation:** Передавать context с timeout.

### 21. `wordWrap` режет UTF-8 байтами (повтор с аудита 3)
- **File:Line:** `internal/tui/messages/messages.go:277-291`
- **Description:** `len(line) > width` и `line[:width]` — байтовые операции.
- **Impact:** Emoji/CJK ломаются.
- **Recommendation:** Использовать `[]rune`.

### 22. Sidebar panic на маленькой ширине
- **File:Line:** `internal/tui/sidebar/sidebar.go:203`
- **Description:** `name[:m.width-len(prefix)-7]` — panic если `m.width < 7`.
- **Impact:** TUI crash на resize.
- **Recommendation:** Bounds guard.

## 🟡 P2 — Средний приоритет (8 штук)

### 23. Empty session ID принимается store

## Общий вердикт

**Score: 3.5/10 → 6.5/10**

Четвёртый аудит обнаружил, что проект страдает от **системного отсутствия defensive programming**. Type assertions без проверок, `panic()` в production path, отсутствие `recover()` для external libraries, полностью сломанный CLI flag, и data-destroying compaction — это не "edge cases", это архитектурные проблемы.

Предыдущие аудиты фокусировались на security и concurrency. Этот аудит показал, что код **не выдержит реального использования**: glamour panic на malformed markdown, OOM на больших сессиях, untyped assertions на каждый keystroke, и безграничный рост memory.

**Волна 3 исправила ~25 issues:** все P0 кроме CVE (требуют обновления toolchain), большинство P1 и P2. Остаются: 4 CVE, DNS rebinding, O(n) rebuild. Для production-ready нужно обновить Go toolchain и рассмотреть архитектурный редизайн TUI rendering pipeline.
