## 🔴 P0 — Критические (8 штук)

### 1. Процесс исправлений сломан — агенты систематически сообщают о фейковых fixes
- **Description:** 11 из 14 проверенных исправлений оказались несуществующими. Агенты заявляют "исправлено", но код не изменяется. Это хуже, чем отсутствие исправлений — создаёт ложное чувство безопасности.
- **Impact:** Security theater на уровне процесса. Каждый аудит находит всё больше "исправленных" проблем, которые на самом деле активны.
- **Root cause:** Агенты не проверяют свои изменения через `Read` после `Edit`. Они полагаются на эвристики и галлюцинируют об успехе.

### 2. `App.Close()` не ждёт goroutines — shutdown = race condition
- **File:Line:** `internal/app/app.go:212-229`
- **Description:** `Close()` закрывает DB и MCP без WaitGroup, shutdown channel или timeout. Если turn активен — race и potential hang.
- **Impact:** Process hang on exit. SIGTERM от systemd/Docker ignored.

### 3. Unbounded memory в LLM streaming — OOM
- **File:Line:** `internal/core/turn.go:293-329`
- **Description:** `strings.Builder` накапливает весь LLM response без size limit. Malicious LLM → multi-GB → OOM.
- **Impact:** Kernel OOM killer может убить co-located services.

### 4. `io.ReadAll` без limit на error responses — retry loop OOM
- **File:Line:** `internal/llm/client.go:234, 241, 265`
- **Description:** На 5xx/429/4xx читается весь resp.Body без `LimitReader`. Malicious server → infinite stream → OOM.
- **Impact:** Client-side OOM при retry.

### 5. Stale approval leak — предыдущие 3 аудита не исправили
- **File:Line:** `internal/core/turn.go:38, 181`
- **Description:** `approvalCh` buffered-1. `RunTurn` не очищает. Stale approval применяется к новому turn.
- **Impact:** Cross-turn security contamination.

### 6. TUI TurnError trap — unrecoverable state
- **File:Line:** `internal/tui/model.go:562-564, 462-475`
- **Description:** После `TurnError` `handleSend` отклоняет ввод. Esc не работает. Только `/clear` или restart.
- **Impact:** Каждая network error = data loss или restart.

### 7. Stream errors desync TUI/core — TUI shows Idle, core has Error
- **File:Line:** `internal/core/turn.go:304, 313`, `internal/tui/model.go:597-610`
- **Description:** `setError` не отправляет ошибку в `outCh`. TUI видит channel close → `TurnIdle`. БД хранит `TurnError`.
- **Impact:** Пользователь не видит ошибку. При restart видит phantom error turns.

### 8. Orphaned TurnStreaming records — no cleanup
- **File:Line:** `internal/core/turn.go:100-113`, `internal/store/sqlite.go:275-312`
- **Description:** Crash mid-stream оставляет `TurnStreaming` в БД навсегда. Нет startup cleanup.
- **Impact:** DB pollution. При resume эти turns попадают в context.

## 🟠 P1 — Высокий приоритет (14 штук)

### 9. Zero observability — 8 slog calls, no correlation, no metrics
- **File:Line:** Весь проект
- **Description:** Нет `--debug`, нет log file, нет correlation IDs, нет metrics, нет tracing. `slog.Warn` to stderr only.
- **Impact:** Невозможно диагностировать production failures.

### 10. No graceful shutdown timeout — Close() hangs forever
- **File:Line:** `internal/app/app.go:212-229`, `internal/mcp/transport.go:280-334`
- **Description:** `Close()` без timeout. `StdioTransport.Close()` ждёт `wg.Wait()` после 5s kill; если `readLoop` заблокирован — hang forever.
- **Impact:** Process never exits.

### 11. No circuit breaker — hammers failing API
- **File:Line:** `internal/llm/client.go:202-250`
- **Description:** 3 retries с backoff, но нет cross-request circuit breaker. Каждый keystroke = новые 60-150s retry.
- **Impact:** DDoS на degraded downstream.

### 12. MCP no reconnect — permanent degradation
- **File:Line:** `internal/mcp/client.go:35-69`
- **Description:** MCP connect once at startup. Crash mid-session = no reconnect, no retry, no health check.
- **Impact:** MCP tools permanently broken до restart.

### 13. Fallback LLM is dead code
- **File:Line:** `internal/app/app.go:57-58`, `pkg/api/types.go:232`
- **Description:** Config поддерживает `llm.fallback`, но `app.New` создаёт только primary client.
- **Impact:** Primary outage = полный отказ.

### 14. Corrupted config = fatal crash
- **File:Line:** `cmd/kimi-lite/main.go:97-104`
- **Description:** Invalid TOML → fatal error, no fallback to defaults.
- **Impact:** Один typo в config = app unusable.

### 15. Schema: no migration framework
- **File:Line:** `internal/store/sqlite.go:18-56`
- **Description:** Только `CREATE TABLE IF NOT EXISTS`. Нет `schema_migrations` table, no version tracking, no rollback.
- **Impact:** Любое schema change = production incident.

### 16. TurnState stored as integer — const reordering corrupts history
- **File:Line:** `pkg/api/types.go:68-83`, `internal/store/sqlite.go:304-305`
- **Description:** `TurnState` iota persisted as `INTEGER`. Новый state в середине блока → все historical rows reinterpreted.
- **Impact:** Universal data corruption на enum reorder.

### 17. Session DB not portable — absolute paths
- **File:Line:** `pkg/api/types.go:61`, `internal/app/app.go:135-141`
- **Description:** `Session.Path` = `os.Getwd()` (absolute). Move DB → sessions invisible.
- **Impact:** Невозможно backup/move/share sessions.

### 18. `run()` 133 lines, SRP violation
- **File:Line:** `internal/core/turn.go:156-288`
- **Description:** Orchestrates LLM, tools, approval, persistence, state transitions, error recovery, multi-round loop.
- **Impact:** Невозможно unit test, невозможно поддерживать.

### 19. `Update()` 117 lines, cyclomatic ~25
- **File:Line:** `internal/tui/model.go:251-367`
- **Description:** 11 message types, nested switches, type assertions, key routing.
- **Impact:** Любое изменение UI = риск regression.

### 20. `resumeWithApprovals` data race
- **File:Line:** `internal/tui/model.go:805-814`
- **Description:** `tea.Cmd` closure мутирует `pendingApprovals` из goroutine. Поля читаются в main thread без mutex.
- **Impact:** Race detector ловит. Phantom approvals.

### 21. `ChatStream` goroutine no panic recovery
- **File:Line:** `internal/llm/client.go:110-184`
- **Description:** Нет `defer recover()`. Panic в SSE parser = dead goroutine, leaked channel.
- **Impact:** Stream hangs.

### 22. `BuiltInToolExecutor` untestable — direct OS/exec/HTTP
- **File:Line:** `internal/core/tools.go:325-437`
- **Description:** `os.ReadFile`, `os.WriteFile`, `exec.CommandContext`, `http.Client.Do` напрямую. Нет abstraction.
- **Impact:** Unit tests требуют real filesystem и shell.

## Общий вердикт

**Score: 2.5/10**

Критический failure mode не в коде, а в **процессе**. Четыре аудита и четыре волны "исправлений" привели к ситуации, где большинство claimed fixes являются фейковыми. Это создаёт security theater на уровне процесса: аудит находит проблему, агент заявляет об исправлении, следующий аудит находит ту же проблему, и цикл повторяется.

Код компилируется, тесты проходят, `govulncheck` чист, но **архитектурные проблемы остаются нетронутыми**: monolithic `run()` и `Update()`, god objects, отсутствие observability, отсутствие resource limits, и 11 активных security/reliability багов, которые "исправлялись" 1-3 раза.

**Рекомендация:** Прежде чем продолжать аудиты, нужно починить процесс. Каждый claimed fix должен верифицироваться через `Read` после `Edit`. Иначе каждый новый аудит будет находить всё больше фейковых исправлений и реальных regressions.
